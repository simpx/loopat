## 快速开始

### 第一步：编写启动脚本

在 `APP-META/docker-config/environment/common/custom/app/bin/` 目录下修改启动脚本：

#### 1. 修改 start.sh（必须）

这是应用的启动入口，容器启动时会自动调用 `start_app()` 函数：

```bash
#!/bin/bash

start_app() {
  echo "start $APP_NAME"
  # 在这里编写你的启动命令
  # 示例：启动 Shell 脚本
  sh $APP_HOME/target/$APP_NAME/src/main.sh
  
  # 示例：启动 Python 应用
  # python3 $APP_HOME/target/$APP_NAME/src/main.py
  
  # 示例：启动 Java 应用
  # java -jar $APP_HOME/target/$APP_NAME/src/app.jar
  
  return 0
}
```

**关键环境变量：**
- `$APP_NAME`：应用名称
- `$APP_HOME`：应用主目录（通常是 `/home/admin`）
- `$APP_HOME/target/$APP_NAME/src/`：你的源码目录
- `$APP_HOME/target/$APP_NAME/conf/`：你的配置文件目录

#### 2. 修改 health.sh（必须）

健康检查脚本，Aone 会定期调用此脚本检查应用是否正常：

```bash
#!/bin/bash

health_check() {
  # 返回0 代表健康检查成功
  # 返回1 代表健康检查失败
  
  # 示例1：检查进程是否存在
  # if ps aux | grep -v grep | grep "your_process" > /dev/null; then
  #   return 0
  # else
  #   return 1
  # fi
  
  # 示例2：检查端口是否监听
  # if netstat -tuln | grep ":8080" > /dev/null; then
  #   return 0
  # else
  #   return 1
  # fi
  
  # 示例3：检查 HTTP 接口
  # if curl -f http://localhost:8080/health > /dev/null 2>&1; then
  #   return 0
  # else
  #   return 1
  # fi
  
  return 0
}
```

#### 3. 修改 setenv.sh（可选）

设置环境变量，在启动前会被加载：

```bash
#!/bin/bash

# 设置 JVM 参数
# export JAVA_OPTS="-Xmx2g -Xms2g"

# 设置 Python 路径
# export PYTHONPATH=$APP_HOME/target/$APP_NAME/src

# 设置其他环境变量
# export MY_ENV_VAR="value"
```

#### 4. 修改 stop.sh（可选）

停止应用的脚本：

```bash
#!/bin/bash

stop_app() {
  echo "stop $APP_NAME"
  # 在这里编写停止逻辑
  # 示例：kill 进程
  # pkill -f "your_process_name"
  return 0
}
```

### 第二步：编写应用代码

在 `src/` 目录下编写你的应用代码：

```bash
src/
├── main.sh          # Shell 脚本
├── main.py          # Python 脚本
├── app.jar          # Java 应用
└── ...              # 其他代码文件
```

**应用代码可以这样访问配置文件：**

```bash
# 在 Shell 中
cat $APP_HOME/target/$APP_NAME/conf/config.json

# 在 Python 中
import os
config_path = os.path.join(os.environ['APP_HOME'], 'target', os.environ['APP_NAME'], 'conf', 'config.json')

# 在 Java 中
String configPath = System.getenv("APP_HOME") + "/target/" + System.getenv("APP_NAME") + "/conf/config.json";
```

### 第三步：配置文件

在 `conf/` 目录下放置配置文件：

```bash
conf/
├── config.json      # 应用配置
├── application.yml  # Spring Boot 配置
└── ...              # 其他配置文件
```

**配置文件特点：**
- 构建时会被打包到镜像中
- 运行时路径为：`$APP_HOME/target/$APP_NAME/conf/`
- 可以在应用代码中通过环境变量读取

## 目录作用说明

### custom/ 目录 - 自定义脚本

**作用：** 存放项目自定义的启动、停止、健康检查等脚本

**重要性：** 所有定制化修改都必须在此目录进行

**包含文件：**
- `start.sh`：定义如何启动应用（必须实现 `start_app()` 函数）
- `stop.sh`：定义如何停止应用（必须实现 `stop_app()` 函数）
- `health.sh`：定义健康检查逻辑（必须实现 `health_check()` 函数）
- `setenv.sh`：定义环境变量

### conf/ 目录 - 配置文件

**作用：** 存放应用的配置文件

**运行时路径：** `/home/admin/{{应用名}}/conf/`

**使用方式：**
```bash
# 通过环境变量访问
$APP_HOME/target/$APP_NAME/conf/config.json

# 实际路径示例（假设应用名为 myapp）
/home/admin/myapp/conf/config.json
```

### src/ 目录 - 源码

**作用：** 存放应用的源代码和可执行文件

**运行时路径：** `/home/admin/{{应用名}}/src/`

**使用方式：**
```bash
# 通过环境变量访问
$APP_HOME/target/$APP_NAME/src/main.sh

# 实际路径示例（假设应用名为 myapp）
/home/admin/myapp/src/main.sh
```

## 常见问题

### Q: 部署后应用无法启动？
**A:** 检查以下几点：
1. 是否修改了 `base` 目录下的脚本？请只在 `custom` 目录下修改
2. `start.sh` 中的 `start_app()` 函数是否正确实现？
3. 启动命令中的路径是否使用了 `$APP_HOME` 和 `$APP_NAME` 环境变量？
4. 应用代码是否放在 `src/` 目录下？

### Q: 如何查看应用日志？
**A:** 应用日志通常在容器的标准输出中，可以通过 Aone 控制台查看。

### Q: 配置文件路径是什么？
**A:** 运行时配置文件路径为 `$APP_HOME/target/$APP_NAME/conf/`，即 `/home/admin/{{应用名}}/conf/`

### Q: 健康检查失败会怎样？
**A:** 健康检查失败会导致容器被标记为不健康，Aone 可能会重启容器或停止流量导入。

### Q: 如何设置 JVM 参数或其他环境变量？
**A:** 在 `custom/app/bin/setenv.sh` 文件中设置环境变量。
